2017，我的年终总结

​ 从2017年1月15日入职，恰指一算也快有12个月，整整一年。适逢白大佬提醒写工作总结，才反应已经快换日历了。个人认为写总结还是很有必要的，不是说为了吹水当谈资，而是写总结，对整个一年的认识做个归纳，对自身的抉择做个评价，也可为新的2018定个小目标。

那些被我支配的项目

一. 公司订餐

​ 第一个拿来练手的小项目， 第一个入PHP坑的项目，也是被捣鼓最多次数的项目。 之前对后端没有一个基本的认知，不了解后端语言，不知道数据库，不懂linux，更不知道PHP技术栈有哪些，职责有哪些。在写项目的同时也是边学习着PHP基础语法，以及各种较为常用的的方法， 踩着mysql数据库的小坑， 也同时了解着常用小组件的原理。

  • 17.02.06 ✔学习了验证码的制作，尝试登录框制作

  • 17.02.07 ✔成功尝试了登录框与数据库之间的交互，复习了下验证码的制作

  • 17.02.08 ✔看下昨天登录框的页面 明天看202页的实例 把登录框给做了

  • 17.02.09 ✔看PDO实现mysql连接

  • 17.02.10 ✔完成了PDO，命名空间的学习。看下PHP编码标准
    。。。

  • 17.02,15 ✔看了登陆状态的实现，了解session的原理，修复了数据库内乱码的问题。下午基本上都是在划水~~

  • 17.02.22 ✔又过了一遍PHP类和对象，粗略看 了下SVN

  • 17.02.23 ✔看了PHP文件上传类的实现准备看PHP GD库里面的缩略图的实现
    。。。

    ​ 以上是当时每天的总结吧，想来应该是这个项目给予了我对后端整个的基础认知吧，大致都是基本了解，也用zend1框架粗略地实现了下订餐，让自己对zend1框架流程有了简单的理解，算是对MVC的一次尝试吧！

二 . bcomm 支付查询功能

​ 参与融资项目的第一个小功能，用zend1实现，虽说只有一个简单的搜索框页面，但是是融资项目的第一次尝试，不免慌得要死，得到开发文档的那天也是让我见到了晚上8点的易盛大厦。

  1. 一个发送请求，然后得到响应保存的小功能，中间加深了http方面的知识。

  2. 因为数据是通过xml传输，实践了下xml的解析与生成。

三. 贷款到期短信提醒功能

​ 通过已有的短信发送接口，对客户以及信贷员发送短信，中间涉及到的表有点多，表关系开始认识得不够清晰。对接接口时卡了一段时间，当时是url编码上的问题，踩了HTTP请求的坑。

  1. 在搜索表单的“到期天数”里输入到期天数(默认5天)来搜索符合该条件的所有提款记录
  2. 通过bootstrap·组件生成弹出框效果，用来显示短信内容以及接受者信息，并计算预估利息。
  3. 将发送的短信信息保存到数据库

四. 担保费管理模块

​ 担保费帐户的操作相关，涉及到数据库事务以及触发器的使用，以及定时任务运行PHP脚本计算应付担保费，东西有点多，也是边写边学的状态。

  1. 通过事务来保证担保费收支记录与担保费帐户中的预收担保费一致，

  2. 通过触发器，让数据库管理担保费帐户操作的历史记录

  3. 通过定时任务运行，计算每日的应付担保费

五. 信贷员短信提醒模块

​ 在贷款到期短信提醒模块的基础上，对信贷员角色授予权限发送短信到企业法人，且可以查看自己客户的短信发送历史情况，进行的比较顺利。

六. 担保费明细

​ 在担保费管理模块的基础上，追加了几个小功能，大致是通过模态框显示担保费计算明细以及历史记录，个人认为的难点在于zend1中的分页组件与ajax的的使用，加深了对分页的理解

七. 上海银行 (还未支配成功)

​ 与黄经理给的域名运维项目同时进行，写域名项目的时候也借鉴了较多的上海银行接口设计思路，对于对接有了个整体的把控。当然，写的时候脑子有点拎不清，写惯了域名部分的regDate，把reqData也下意识地写成了regData，着实有点坑队友的意思。

  1. 根据接口文档建立数据表。

  2. 生成相应的数据，并保存到request表

  3. 发送到签名服务器，生成相应的签名，

  4. 将签名中的所需数据发送到上海银行接口，拿到响应值，解析保存到数据库

今年翻过的书

​ 基本是技术书籍，没时间翻看其他领域的知识，只能中午偶尔知乎日报补补其他领域知识。

  • 《PHP和SQLWeb开发》：各大应用中被吹捧的“PHP圣经”, 内容主旨跟书名一样，主要讲得PHP跟MYSQL 两方面，以及宽泛地介绍了后端的知识点，著作时间久远，但知识点都有带到，很不错的入门书。

  • 《图解HTTP》: 插图很多，可以较为轻松地了解HTTP协议， 相对《HTTP权威指南》,这本可能更适合我。

  • 《HeadFirst设计模式》：类似与《图解HTTP》风格，看它主要想在思想上有设计模式的概念，可惜自身代码量不够，功力不深，不能很好地理解。

  • 《编写可维护的JavaScript》：应该是现阶段对我而言最有食用价值的书籍，细致地讲了代码风格以及编程规范，详细地阐述了为什么要这么做，以及作者之前因为没有制订规范所产生的后果，很有帮助，毕竟码代码不是一个人的时，码得省心省力，便于协调，那便是极好的

  • 《深入PHP：面向对象、模式与实践》：很好地阐述了面向对象思想，知识点很多- -，属于小白发现新大陆的状态，虽然相对《设计模式》写的较为笼统，但对php面向对象帮助很大。

  • 《大话数据结构》：数据结构入门书， 但我可能还没入门。

  • 《深入浅出MySQL++数据库开发、优化与管理维护》：很不错的mysql教科书，应付日常业务开发感觉足够。

  • 《快乐的linux命令行》：鸟哥的私房菜的厚度以及深度 让我望而生畏，取而代之的一本linux书籍。

  • 以及一些杂七杂八的电子书。。。

得与失

一. 得

  • 将近一年的后端环境，让我对后端整个基本流程有个大致的认知，涉及到的诸如编程语言相关（PHP）, linux相关，数据库相关有很大的提升。
  • 技术层面，代码风格以及编码规范上有清晰的明确，脑子里有了一只手能数出来的设计模式思想，从之前的面向过程编程稍稍进入了点面向对象思想领域（毕竟框架MVC)。但很多细节方面都还没踩过坑，以为自己懂，但没发现自己的问题所在。安全方面，提升了防范XSS，sql注入，CSRF的意识（其他诸如文件上传等等还没实际尝试）
  • 运势层面，本命年运势强盛，入社遇贵人，顶头上司白大佬授人以渔的指导，加之部门小组有代码审核，简直是业界良心。
  • 沟通层面，从一个社交恐惧症患儿转为了能勉强正常沟通。
  • 健康层面，得到了20斤体重

二. 失

  • 从前端noob转向了后端noob，这一年一直在宽泛地去学习各种后端知识体系，想着能先上手完成给予的业务，没有特意去深挖某个领域。可能导致学到的是比较浅显的知识，编程遗留潜在bug较多。

  • 一直专注于技术，导致沟通水平下降，表达不清晰。

  • 归家后的时间少，但还停留在大学的生活状态， 但运动量没有跟上去，晚上睡的晚，导致作息不好，身体健康有所下降。

  • 脱落了一些毛发

存在的一些问题

  1. 技术上可能为了赶时间，没有深究其中涉及的原理，只做了一些简单的记录，妄想着闲下来的时候回去再重头深挖前面碰到的疑难点，反而得到的结果是印象笔记收集了一大堆，真正看懂，熟练掌握的并不多。加之，碎片化阅读导致提升效率不高，应考虑改善时间管理方向，精简问题，不可囫囵吞枣式阅读。
  2. 当一个项目完完整整地交付到手中时，会手足无措，不知道各个细节点人员的责任划分，以及整个项目从该如何听取需求，了解该项目所属领域知识，设计表结构，到统一编码风格，人员沟通等等一系列开发过程中所需的流程都没有做得很老道，显得颇为稚嫩，还需通过实践以及投入时间来给予提升。
  3. 英语水平欠缺还需提升，读方面还算及格，能勉强阅读stackoverflow，github上的一些问题和类似于readme之类的说明， 但口语听说方面欠缺，英语仅仅是单词交流上都可能存在影响。改进方案大致是将自己融入英语环境中，所有智能设备能英语全英语。
  4. 自我约束能力不够，有拖延症，导致作息不正常，影响第二天工作状态。每天需适量运动，增强自信。

最后

​ code review 这种程序员交流提升的活动，请务必继续下去！其他我已经很满意了，不再奢求什么了。
